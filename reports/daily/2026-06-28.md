- Rebuilt and restarted the formal container service on port `3000`.
- Confirmed the API endpoint `/api/mutation-analysis/genes` returned `200 OK`.

### 3. Replaced Placeholder Protein Graphic With Real 3D Structure Viewer

The user pointed out that the protein area was still wrong because it looked like a placeholder instead of the 3D protein effect shown on the AlphaFold website.

Completed implementation:

- Added `ngl` as a website dependency.
- Served the NGL bundle through `/vendor/ngl.js`.
- Reworked the protein structure card into an interactive 3D viewer.
- Added controls for reset, full screen, and spin behavior.
- Used AlphaFold-style pLDDT coloring. pLDDT is a confidence score for predicted protein structure positions.
- Updated frontend structure loading so selecting a mutation can load the relevant protein structure.

Plain explanation:

- NGL is a JavaScript library for showing protein structures in 3D.
- AlphaFold predicts protein structures from protein sequences.
- A 3D viewer is useful because mutation positions can be connected to the physical shape of the protein, not only to a table row.

### 4. Added Structure Proxy And Fragment Support For Large Proteins

Large proteins such as APC and TTN are too big to load as one full model in a normal web viewer. The implementation was changed to load RCSB computed AlphaFold fragments instead.

Completed backend work:

- Added a read-only protein structure endpoint:

```text
GET /api/mutation-analysis/proteins/:uniprotId/structure
```

- Added support for both:
  - PDB files from AlphaFold DB
  - BCIF files from RCSB computed AlphaFold models

- Added server-side structure caching.
- Added response headers describing:
  - structure format
  - RCSB structure entry
  - fragment start
  - fragment end
  - selected residue position inside the fragment

Important verified examples:

- `TTN p.R25363C` loads `AF_AFQ8WZ42F127`
- TTN fragment range: `25,201-26,600`
- TTN selected residue maps to fragment-local position `163`
- `APC p.R1450*` loads `AF_AFP25054F8`
- APC fragment range: `1,401-2,800`

Plain explanation:

- A fragment is one smaller piece of a very large protein.
- BCIF is a compressed structure format that is better for web loading than very large text-based PDB files.

## What Was Learned

- A page can look updated in the browser while the real container backend is still misconfigured. API checks and browser checks both matter.
- PostgreSQL connection errors can be caused by missing container environment variables, not missing data tables.
- Very large proteins should not be loaded as full structures in a web viewer. Fragment loading is a more practical design.
- RCSB computed AlphaFold fragments can make APC and TTN structure viewing possible in a browser.
- Showing every mutation label is not useful for very large genes. Drawing every point but labeling only selected or important mutations gives a clearer result.
- Browser validation is necessary for scientific visualizations because API success alone does not prove that charts, 3D canvases, scrolling, and drawers work correctly.

## Issues And Limitations

- The structure viewer uses predicted structures, not experimentally confirmed structures for every protein.
- For very large proteins, the viewer shows the relevant fragment rather than the whole protein.
- Some mutation records cannot be placed on the protein coordinate map because they do not have a clear amino acid position.
- The page is for research and education only. It is not a medical diagnosis or treatment recommendation tool.
