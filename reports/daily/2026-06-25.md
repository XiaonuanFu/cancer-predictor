  - first model used TCGA plus UCSC Xena data

  - Logistic Regression
  - Random Forest
  - Linear SVM

- Do not use three separate View buttons for:
  - Metrics
  - Confusion matrix
  - Important features

- Instead, show all three result types together in one continuous comparison view.

Machine learning means training a program to find patterns from data. Batch effect means a technical difference caused by data being produced in different experiments, labs, or processing pipelines.

## What Was Learned

- Local website changes are not enough for this project. The container-served website uses `docker_storage/jupyter/node-app`, so web updates must be synced there.
- The Model page is clearer when the three machine learning models are compared side by side instead of hidden behind View tabs.
- A perfect score, such as `100%` accuracy, is not always a good sign. It can mean the model learned data-source differences instead of true biology.
- For a personal research website, a small GitHub and email footer can make the site feel more complete without adding visual clutter.
- Route redirects help keep old links usable after website navigation changes.

---

## Issues / Limitations

- Some web files had existing uncommitted changes before today's work, especially styling files.
- Container and local versions can drift if sync is skipped.
- Some verification used HTTP/content checks when full browser tooling was unavailable in that environment.
- The website is still a research and education project, not a medical diagnosis tool.
