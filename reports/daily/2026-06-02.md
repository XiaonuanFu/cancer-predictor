## Conversation 2: Penguin Predictor Notebooks Under `ml_study`

## Work Completed

- Used the local penguin CSV dataset.
  - Source data: `data/ml_data/penguins.csv`.
  - Container data path: `/workspace/data/ml_data/penguins.csv`.
  - Target label: `species`.
  - Features include island, bill measurements, flipper length, body mass, sex, and year.

- Created a new penguin predictor notebook project.
  - Local output directory:
    - `ml_study/penguin_predictor/`
  - Jupyter container output directory:
    - `/workspace/ml_study/penguin`
  - Host-mounted container copy:
    - `docker_storage/jupyter/ml_study/penguin/`

- Generated four notebooks.
  - `00_penguins_eda.ipynb`
    - Performs exploratory data analysis.
    - Shows data shape, column types, missing values, species distribution, feature distributions, and feature relationships.
  - `01_logistic_regression_penguins.ipynb`
    - Trains Logistic Regression to predict penguin species.
    - Uses a preprocessing pipeline with numeric imputation/scaling and categorical one-hot encoding.
  - `02_random_forest_penguins.ipynb`
    - Trains Random Forest to predict penguin species.
    - Shows model evaluation and feature importance.
  - `03_svm_penguins_model_comparison.ipynb`
    - Trains SVM and compares Logistic Regression, Random Forest, and SVM on the same train/test split.
    - Reports accuracy, macro precision, macro recall, macro F1, and confusion matrices.

- Standardized the modeling workflow.
  - Used `train_test_split(..., stratify=y, random_state=42)`.
  - Used the same train/test split across models.
  - Used `ColumnTransformer`:
    - Numeric features: median imputation; Logistic Regression and SVM also use scaling.
    - Categorical features: most-frequent imputation plus one-hot encoding.
  - Model settings:
    - Logistic Regression: `LogisticRegression(max_iter=2000, multi_class='auto')`
    - Random Forest: `RandomForestClassifier(n_estimators=200, random_state=42)`
    - SVM: `SVC(kernel='rbf', probability=True, random_state=42)`

- Corrected the output directory after user feedback.
  - The first implementation placed files under `ml_data`, following the initial plan.
  - The user clarified that both local and container notebook code should be under `ml_study`.
  - Moved the local files to `ml_study/penguin_predictor/`.
  - Moved the Jupyter container files to `/workspace/ml_study/penguin`.
  - Removed the incorrect `ml_data` output directory created during the first attempt.
