## Conversation 1: Fix COAD PanCanAtlas Mutation Notebook

## Problem

The notebook `coad_pancanatlas_mutation_landscape_report.ipynb` failed in step 3 with:

```text
NameError: name 'read_sql' is not defined
```

The cause was that `read_sql` was defined in the first setup cell. If the notebook kernel was restarted, or if step 3 was run directly, the helper function no longer existed in memory.

- Added safer setup logic:
  - later SQL cells now check whether the setup cell has run
  - if setup has not run, the notebook shows a clearer bilingual instruction instead of failing with a raw Python `NameError`

- Fixed the generator output path so regenerated notebooks target the current project structure:
  - `py_code/coad-predictor-model/reports/...`

## Second Notebook Error

After the first fix, the setup cell failed with:

```text
RuntimeError: Please set PGPASSWORD before running this notebook.
```

The issue was that the Jupyter container did not provide `PGPASSWORD` as an environment variable. Other project notebooks already had a fallback path, but this mutation notebook did not.

## Password Fallback Fix

- Updated setup logic to read the database password in this order:
  - first from environment variables such as `PGPASSWORD` or `DB_PASSWORD`
  - then from the project config file:
    - `py_code/coad-predictor-model/src/config.py`

- This makes the notebook behave more like the other COAD report notebooks.

## Jupyter Mounted Notebook Fix

The user was still seeing the old setup cell because Jupyter was opening the mounted workspace copy, not the local `py_code` copy.

- Synced the fix into the actual Jupyter-mounted notebook:
  - `docker_storage/jupyter/coad-predictor-model/reports/coad_pancanatlas_mutation_landscape_report.ipynb`

- Confirmed the mounted notebook now contains the newer setup logic:
  - `from src import config as project_config`
  - `DB_PASSWORD = get_db_setting(("PGPASSWORD", "DB_PASSWORD"), "DB_PASSWORD")`

## What Was Learned

- Jupyter can keep an old notebook cell in the browser even after the file on disk changes. Closing and reopening the notebook tab may be needed to load the new code.
- A notebook that depends on setup state should fail with a clear instruction, not a raw `NameError`.
- Website page counts must be separated by context. The COAD Data report page uses report-level counts, while the model page can use different model-training counts.
- ECharts works well for many dashboard-style charts, but specialized biology charts may need biological visualization libraries or notebook screenshots.
- Mobile chart design sometimes needs a different layout, not only a smaller version of the desktop chart.

---
