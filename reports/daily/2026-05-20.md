  - Key table row counts:
    - `chembl.activities`: 24,267,312

    - `chembl.compound_structures`: 2,854,815
    - `chembl.molecule_dictionary`: 2,878,135

## Issues / Notes

- Docker Hub network access was unstable, and image metadata requests timed out during builds.
  - Used `BASE_IMAGE=bio:latest` to build from the existing local image and avoid blocking on Docker Hub.

- The ChEMBL official dump failed on the `idx_assays_desc` btree index because long text exceeded PostgreSQL's btree row size limit.
  - No data was lost.
  - Replaced it with a same-name hash index:

```sql
CREATE INDEX idx_assays_desc ON chembl.assays USING hash (description);
```

- Started importing local TCGA data into PostgreSQL.
  - User-provided directory was `data/rawdata`; actual local directory was identified as `data/raw_data`.
  - Target database: `bio`.
  - Target schema: `bio_tcga`.
  - Added import script: `scripts/import_tcga_raw_data.py`.

- Added import script capabilities.
  - Direct tables are created automatically from headers with columns initially stored as `text`.
  - Clinical, manifest, MAF, segment, and sample annotation files use direct table imports.
  - RNA, methylation, miRNA, and other ultra-wide matrices are split into sample metadata tables and matrix value tables to avoid PostgreSQL column-count limits.
  - Added encoding cleanup for non-UTF-8 dirty bytes in the clinical file.
  - Added `import_manifest` to track source file, load mode, columns, and notes.

- Completed or advanced several TCGA imports.
  - GDC manifest.
  - miRNA sample information.
  - RPPA pancan clean.
  - ABS segment/table.
  - SNP6 segment.
  - Clinical patient followup.
  - MC3 public MAF.
  - RNASeq gene expression matrix.
  - HumanMethylation450 beta matrix was started and the import approach was optimized multiple times.

## Issues / Notes

- `data/` is ignored by `.gitignore`, so normal `rg --files` does not list local raw data directories. Use `find` when inspecting ignored local data.

- The TCGA methylation matrix is extremely large.
  - `jhu-usc.edu_PANCAN_HumanMethylation450.betaValue_whitelisted.tsv` is about 39 GB.
  - The initial `double precision[]` parsing approach was too slow, so it was changed to a raw text vector approach.
  - The next bottleneck was `docker exec` stdin streaming, so the script was changed to start a temporary PostgreSQL client container, mount `data/raw_data` read-only, and run `awk | psql COPY` inside the container network.

- The TCGA import conversation was interrupted after optimizing the largest matrix import path.
  - Small tables and some large tables had already been imported.
  - The largest methylation matrix and full final validation still needed follow-up.
