## Conversation 1: Research Scope, GDC COAD Data, and Database Loading

## Work Completed

- Clarified the research project direction.
  - The original idea was to infer cancer likelihood from existing cancer sequences.
  - The scope was narrowed to a more scientifically grounded and practical COAD colon cancer `tumor vs normal` classification task.
  - Recommended not starting from FASTQ/BAM workflows in the first version, and instead using processed expression matrices or STAR counts.
  - Clarified that the model is research-only and should not be described as a clinical diagnostic model or a future cancer risk predictor for healthy individuals.

- Discussed normal sample data sources.
  - TCGA/GDC `Solid Tissue Normal` can be used as adjacent normal tissue.
  - GTEx / UCSC Xena can be used later as an expanded normal data source.
  - The first version should prioritize TCGA-COAD `Primary Tumor` and `Solid Tissue Normal` samples to reduce batch-effect risk.

- Downloaded GDC TCGA-COAD STAR counts data.
  - Data type: `Transcriptome Profiling / Gene Expression Quantification / STAR - Counts`.
  - Download directory: `data/gdc_tcga_coad_star_counts/`.
  - `Primary Tumor`: 481 files, about 1936.3 MiB.
  - `Solid Tissue Normal`: 41 files, about 165.5 MiB.
  - Total: 522 files.
  - Saved manifests, metadata, and direct download links.

- Created and loaded the `tcga_coad` schema.
  - Target database: `bio` in the local `bio-postgres` container.
  - Added import script: `scripts/import_tcga_coad_star_counts.py`.
  - Created the `tcga_coad` schema.
  - Imported 522 GDC file metadata records.
  - Imported 60,664 gene annotation records.
  - Imported the `star_gene_counts` long table with 31,666,608 rows.
  - Created the `tcga_coad.star_counts_with_metadata` view to join counts, gene annotations, and GDC file metadata.

- Defined sample filtering logic for `bio_tcga` and `tcga_coad`.
  - COAD tumor samples in `bio_tcga` are selected by joining to `tcga_cdr_tcga_cdr.type = 'COAD'`.
  - TCGA barcode positions 14-15 code `01` means `Primary Solid Tumor`.
  - TCGA barcode positions 14-15 code `11` means `Solid Tissue Normal`.
  - COAD RNA sample counts in `bio_tcga.matrix_rnaseq_gene_expression_samples`:
    - all COAD samples: 492
    - Primary Tumor (`01`): 449
    - Solid Tissue Normal (`11`): 41

- Updated the requirements document.
  - Updated `doc/requirements/cancer-prediction-model-requirements.md`.
  - Documented first-version model scope, data sources, task definition, limitations, and the future Xena/GTEx extension path.

## Issues / Notes

- The "522 records" in `tcga_coad` refers to 522 GDC files, not 522 database tables.
- First-version normal features use `tcga_coad.star_counts_with_metadata.tpm_unstranded`, with `log2(TPM + 1)` applied before modeling.
- First-version tumor features use `bio_tcga.matrix_rnaseq_gene_expression` and `bio_tcga.matrix_rnaseq_gene_expression_samples`.
- Tumor and normal samples come from different schemas/pipelines, which must be treated as a major model interpretation limitation.

## Issues / Notes

- Early training exposed mixed whitespace and a few non-standard numeric tokens in `bio_tcga.matrix_rnaseq_gene_expression.values_text`; tolerant parsing was added.
- The first training run found missing values in the merged matrix; all three model Pipelines now use `SimpleImputer(strategy="median")`.
- One sync attempt used `rsync --delete` and removed generated `data/` outputs in the Jupyter runtime directory. The workflow was corrected to sync code, rerun `prepare`, then rerun `train`.
- Model performance is very high, but because tumor and normal data come from different schemas/pipelines, the report explicitly states that the model is exploratory research and not a clinical diagnostic model.
