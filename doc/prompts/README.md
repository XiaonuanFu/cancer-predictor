# Prompt Directory

Use this directory to record and organize prompts used during daily project work.

## Purpose

- Keep a daily history of prompts used for coding, analysis, documentation, data work, and research.
- Preserve useful prompts so they can be reused or improved later.
- Track which prompts produced good results, partial results, or need revision.

## Daily Workflow

At the end of each work session, create or update a daily prompt log:

```text
doc/prompts/YYYY-MM-DD.md
```

For each prompt, record:

- Original prompt text.
- Task category.
- Context or files involved.
- Result summary.
- Whether the prompt should be reused, revised, or retired.

## Suggested Categories

- `requirements`
- `web-code`
- `database`
- `data-analysis`
- `docker-environment`
- `documentation`
- `research`
- `debugging`
